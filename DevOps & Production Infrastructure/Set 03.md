# DevOps & Production Infrastructure: Set 3
## CI/CD Pipelines, GitOps & Zero-Downtime Deployments

##  10 Conceptual Questions
1. **CI vs. Delivery vs. Deployment:** What is the exact difference between Continuous Integration, Continuous Delivery, and Continuous Deployment? Why do many mature teams intentionally stop at "Delivery" (manual production trigger) rather than fully automated "Deployment"?
2. **Zero-Downtime Laravel Deploys:** How does the "symlink swap" deployment strategy work in Laravel? Why is atomic deployment critical, and what happens to in-flight HTTP requests or running queue jobs exactly at the moment the `current` symlink changes?
3. **Database Migrations in Pipelines:** Why are DB schema changes the single highest risk in zero-downtime deployments? Explain the "expand and contract" pattern, and why you should never write a migration that breaks backward compatibility with the currently running code.
4. **Build Once, Deploy Many:** Why is it an anti-pattern to rebuild your Docker image or compile assets separately for Dev, Staging, and Production? How do you pass the exact same artifact through environments while injecting environment-specific config?
5. **Secrets Management & OIDC:** Why is storing long-lived cloud credentials in CI/CD variables dangerous? Explain how OpenID Connect (OIDC) allows your pipeline to authenticate with AWS/GCP/GitHub Container Registry temporarily and securely without storing passwords.
6. **GitOps vs. Push-Based CI/CD:** How does GitOps (e.g., ArgoCD, Flux) invert the traditional deployment model? Explain the "pull-based" architecture where the cluster syncs its state from a Git repo, and why this eliminates configuration drift.
7. **Rollback Strategies:** What is the difference between rolling back *code* (reverting git, swapping symlink, deploying previous image) vs rolling back *database* changes? Why is database rollback usually impossible, and how do you design forward-only migrations?
8. **Flaky Tests in CI:** Why is a flaky test (passes sometimes, fails randomly) worse than a consistently failing test in a deployment pipeline? How do you handle quarantine, retry logic, and state isolation to maintain team trust in the CI?
9. **Deployment Gates & Health Checks:** What is the difference between "the deployment script finished" and "the deployment was successful"? How do you use automated health checks, error-rate monitoring, and canary analysis to trigger an automatic rollback if things go wrong?
10. **Pipeline Caching & Artifacts:** How do you optimize slow CI pipelines? Explain the difference between caching dependencies (composer/npm) and passing build artifacts (compiled JS, Docker layers) between pipeline stages.
