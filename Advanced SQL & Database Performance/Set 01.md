# Advanced SQL & Database Performance: Set 1
## Query Execution, Indexing Strategies & The Query Optimizer

## ­¤ōś 10 Conceptual Questions
1. **How B-Tree Indexes Actually Work:** Why are B-Trees the default for almost all relational databases? Explain the difference between a Clustered Index (Primary Key) and a Non-Clustered (Secondary) Index, and why secondary indexes require a "bookmark lookup" (or row ID lookup).
2. **The Cost-Based Query Optimizer:** How does the database decide whether to use an index, do a full table scan, or use a different index? Why does the optimizer sometimes choose the "wrong" index, and how do statistics influence this decision?
3. **Composite Indexes & The Left-Most Prefix Rule:** If you have a composite index on `(status, created_at)`, why can the database use it for `WHERE status = 'active'` but *not* for `WHERE created_at > '2023-01-01'`? How does the order of columns in a composite index dictate its usefulness?
4. **Covering Indexes & "Using index":** What does it mean for an index to be "covering"? Why does seeing `Using index` in the `EXPLAIN` output mean the database didn't even touch the actual table data, and how do you design for this?
5. **Cardinality & Selectivity:** What is the difference between high cardinality (e.g., `email`) and low cardinality (e.g., `is_active`)? Why do databases often ignore indexes on low-selectivity columns, and when is a full table scan actually faster?
6. **Implicit Type Casting:** Why does querying a string column with an integer (e.g., `WHERE phone_number = 12345`) destroy index usage? Explain how implicit conversions force the database to evaluate every row.
7. **`EXPLAIN` vs. `EXPLAIN ANALYZE`:** What is the difference between the estimated execution plan (`EXPLAIN`) and the actual execution plan (`EXPLAIN ANALYZE` / `EXPLAIN EXECUTE`)? Why are actual row counts and execution times crucial for debugging?
8. **The Hidden Cost of `SELECT *`:** Beyond fetching unnecessary data over the network, how does `SELECT *` hurt the database engine's memory (Buffer Pool / Shared Buffers) and prevent the use of covering indexes?
9. **Correlated Subqueries vs. JOINs:** Why do correlated subqueries (a subquery that references the outer query) often result in O(N) execution times? How does rewriting them as `JOIN`s or `EXISTS` change the execution plan?
10. **Index Bloat & Maintenance:** What causes an index to become "bloated" or fragmented over time? How do heavy `UPDATE` and `DELETE` operations degrade index performance, and when should you rebuild or reorganize them?
