# TypeScript Programming Language: Set 2
## Advanced TypeScript: Type-Level Programming, Metaprogramming, and Complex Patterns

## 📘 10 Conceptual Questions

1. **Conditional Types & The `infer` Keyword:** How do Conditional Types (`T extends U ? X : Y`) allow you to create types that act like `if/else` statements at compile time? How does the `infer` keyword allow you to "extract" and assign a new type variable from within another type (e.g., extracting the resolved type of a `Promise` or the return type of a function)?
2. **Mapped Types & Key Remapping:** How do Mapped Types (`{ [K in keyof T]: ... }`) allow you to iterate over the keys of an existing type to create a new one? How does the `as` clause in mapped types (Key Remapping) let you dynamically filter out keys or rename them based on their original names?
3. **Template Literal Types:** How do Template Literal Types (e.g., `` `on${Capitalize<string>}` ``) combine with Union types to generate massive sets of highly specific string literals? How does this enable type-safe event emitters, CSS-in-JS class names, or strict routing paths?
4. **The `satisfies` Operator (TS 4.9+):** How does the `satisfies` operator differ from a standard type annotation (`:`)? Why is it crucial for validating an object against a broad type (like `Record<string, string>`) while *preserving* the exact, narrow literal types of its individual properties?
5. **Variance: Covariance, Contravariance, and Bivariance:** When dealing with functions and inheritance, how does TypeScript's structural typing handle variance? Why are function parameters *contravariant* (meaning a function expecting a `Dog` cannot be assigned where a function expecting an `Animal` is expected), and how does this prevent runtime errors?
6. **Declaration Merging & Module Augmentation:** How does TypeScript allow you to extend existing interfaces or modules without modifying their original source code? How is this used to add types to global objects (like `Window`) or augment third-party libraries (like adding custom properties to an Express `Request` object)?
7. **Type-Level Recursion & Tail Recursion:** How can you write recursive conditional types (e.g., to deeply make all nested properties `Readonly`, or to parse a string type character by character)? What causes the "Type instantiation is excessively deep" error, and how does TypeScript optimize tail-recursive conditional types to prevent it?
8. **Advanced Generic Constraints (`extends` & `keyof`):** How do you restrict a generic type parameter to ensure it has a specific property? How do you use `keyof` to constrain a generic so that it can only be a valid key of another object, creating perfectly type-safe dictionary lookups?
9. **`declare`, `.d.ts` files, and Ambient Context:** What is the purpose of declaration files (`.d.ts`)? How do you use the `declare` keyword and `declare global` to tell TypeScript about variables, functions, or modules that exist at runtime (e.g., injected by a CDN script or Webpack) but aren't defined in your actual TS code?
10. **TypeScript Decorators (Legacy vs. Stage 3):** How do TypeScript's experimental decorators (using `reflect-metadata`) differ from the new, standardized ECMAScript Stage 3 decorators? How are they used in frameworks like NestJS or Angular for dependency injection, method wrapping, and metadata reflection?
