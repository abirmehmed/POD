# Security Engineering: Set 2
## Authentication, Authorization & Identity Management

## 📘 10 Conceptual Questions
1. **Session vs. Token (JWT) Authentication:** What are the fundamental trade-offs between stateful sessions (cookies) and stateless tokens (JWT)? Why is JWT often misused as a "silver bullet," and why do frameworks like Laravel recommend stateful cookies (Sanctum) for SPAs over JWT?
2. **OAuth 2.0 vs. OpenID Connect (OIDC):** What is the critical difference between OAuth 2.0 (an authorization framework) and OIDC (an authentication layer on top of OAuth 2.0)? Why should you never use an OAuth 2.0 access token to identify a user?
3. **Password Hashing Mechanics:** Why are fast hashing algorithms (MD5, SHA-1, SHA-256) completely obsolete for passwords? Explain how Argon2id and Bcrypt use work factors (cost) and salts to defeat brute-force and rainbow table attacks. What is a "pepper," and when is it useful?
4. **Session Fixation & Hijacking:** How does an attacker force a victim to use a known session ID (fixation), and how do they steal it (hijacking)? Why is calling `session()->regenerate()` immediately after a successful login the primary defense against fixation?
5. **RBAC vs. ABAC:** Compare Role-Based Access Control (RBAC) and Attribute-Based Access Control (ABAC). When does a simple `hasRole('admin')` check become insufficient, requiring ABAC (e.g., "User can edit this document only if they are the owner AND the document status is 'draft'")?
6. **Multi-Factor Authentication (MFA) Realities:** How does Time-based One-Time Password (TOTP) actually work under the hood? Why is SMS-based 2FA considered vulnerable (SIM swapping, SS7 attacks), and why are recovery codes a mandatory fallback?
7. **Password Reset Flow Vulnerabilities:** What are the common pitfalls in password reset implementations? Explain token expiration, timing attacks (enumerating valid emails via response time), and the danger of sending reset links via HTTP instead of HTTPS.
8. **JWT Security Pitfalls:** If you *must* use JWTs, what are the critical security checks? Explain the `alg: none` vulnerability, the importance of verifying the signature and `iss`/`aud` claims, and why you should never store sensitive data (like PII) in the payload.
9. **Principle of Least Privilege in APIs:** How do you prevent an over-privileged token from doing damage if compromised? Explain token scoping/abilities (e.g., Laravel Sanctum's `token-can('posts:write')`) and how to enforce it at the route level.
10. **Email Enumeration Prevention:** How can an attacker map your entire user database by observing the differences in "User not found" vs. "Reset link sent" responses? How do you design endpoints to return generic messages and use constant-time comparison to prevent timing attacks?
