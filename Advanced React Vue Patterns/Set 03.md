# Advanced React/Vue Patterns: Set 3
## Performance Optimization & Advanced Rendering

## 📘 10 Conceptual Questions
1. **Reactivity vs. Virtual DOM Diffing:** How does Vue’s fine-grained reactivity (Proxy traps) fundamentally differ from React’s Virtual DOM diffing algorithm? Why does a state change in React potentially re-render an entire subtree, while Vue only updates the specific DOM node that depends on that state?
2. **The Cost of Memoization:** Why is indiscriminately wrapping everything in `useMemo`/`useCallback` (React) or `computed` (Vue) an anti-pattern? Explain the overhead of dependency array comparison and closure creation, and when memoization actually *degrades* performance.
3. **List Virtualization (Windowing):** Why does rendering 10,000 DOM nodes crash or freeze a browser, even if the data is already in memory? How do virtualization libraries solve this by calculating viewport bounds and only rendering visible items?
4. **Advanced Code Splitting:** How does a bundler (Vite, Webpack, Rollup) decide what goes into a chunk? Explain the difference between route-level splitting, component-level splitting, and vendor chunking, and how to prevent a single massive `app.js` file.
5. **Concurrent Features (React) vs. Non-Blocking Updates (Vue):** How does React’s `useTransition` allow high-priority updates (like typing) to interrupt low-priority updates (like filtering a large list)? How does Vue achieve similar non-blocking behavior through its scheduler and `nextTick`?
6. **Web Workers for CPU-Bound Tasks:** Why should heavy computations (e.g., sorting 100,000 items, complex image manipulation) never run on the main thread? How do you safely communicate between the main thread and a Web Worker without blocking the UI?
7. **Streaming SSR & Selective Hydration:** How do modern frameworks (Next.js, Nuxt) improve Time to First Byte (TTFB) and Time to Interactive (TTI) by streaming HTML in chunks and hydrating components progressively, rather than waiting for the entire page to be ready?
8. **Memory Leaks in SPAs:** How do Single Page Applications leak memory over time? Explain the dangers of uncleared `setInterval`s, un-removed `window.addEventListener`s, and state updates on unmounted components.
9. **Core Web Vitals (CWV) Mapping:** How do architectural decisions directly impact Largest Contentful Paint (LCP), Interaction to Next Paint (INP), and Cumulative Layout Shift (CLS)? Why is optimizing for CWV a business metric, not just a technical one?
10. **Bundle Analysis & Tree-Shaking Failures:** What causes a library to fail tree-shaking (e.g., CommonJS modules, side-effectful code)? How do you read a bundle analyzer report to identify duplicate dependencies or bloated vendor chunks?
