# Advanced React/Vue Patterns: Set 2
## Advanced Component Composition & Headless UI Architecture

## 📘 10 Conceptual Questions
1. **The Compound Component Pattern:** Why is passing 10+ boolean props to a single component (e.g., `<Modal showHeader={true} showFooter={false} ...>`) an anti-pattern? How does the Compound Component pattern (e.g., `<Modal><Modal.Header>...</Modal.Header></Modal>`) solve this using implicit state sharing?
2. **Render Props (React) vs. Scoped Slots (Vue):** How do these two concepts solve the exact same problem? Explain how they enable "Inversion of Control" by allowing a parent component to dictate *what* is rendered, while the child component dictates *when* and *with what data* it is rendered.
3. **Headless UI Philosophy:** What does it mean for a component to be "headless"? Why is separating behavior/logic (keyboard navigation, focus management, ARIA attributes) from presentation (CSS/styling) the gold standard for building scalable, accessible design systems?
4. **Context API Re-render Traps:** Why does updating a value in a React Context or Vue `provide` cause *every* consuming component to re-render, even if they only use a small part of the state? How do you mitigate this (e.g., splitting contexts, memoizing providers, or using selector patterns)?
5. **Polymorphic Components:** How do you build a component that can render as different HTML elements based on a prop (e.g., `<Text as="h1">` or `<Button as="a" href="/link">`) while maintaining strict TypeScript type safety and forwarding the correct DOM attributes?
6. **Custom Hooks/Composables for Side Effects:** Why should logic like `window.addEventListener`, `IntersectionObserver`, or `setInterval` be extracted into custom hooks/composables? What are the strict rules for cleanup to prevent memory leaks in SPAs?
7. **Manipulating Children:** When is it appropriate to inspect or manipulate `children` (React's `React.Children.map` or Vue's `$slots`)? Why is this generally discouraged in favor of explicit props or compound patterns, and what are the edge cases where it's necessary (e.g., building a `<Select>` or `<RadioGroup>`)?
8. **Forwarding Refs:** Why can't you just pass a `ref` to a functional component or a custom Vue component by default? Explain the mechanics of `forwardRef` (React) or `defineExpose` / `$attrs` (Vue) to expose underlying DOM nodes to parent components.
9. **Portals / Teleport:** Why do Modals, Tooltips, and Dropdowns often break when placed inside parents with `overflow: hidden` or specific `z-index` contexts? How do Portals (React) or `<Teleport>` (Vue) solve this by rendering the DOM node outside the normal React/Vue tree hierarchy?
10. **The YAGNI Limit of Composition:** When does advanced composition (Compound components, render props, heavy custom hooks) cross the line into over-engineering? What signals indicate that a simple component with a few props is actually the better architectural choice?
