4. **State Sharing in Micro-Frontends:** If you have three independently deployed micro-frontends on one page, how do they share global state (like authentication or theme) without tight coupling or race conditions? (Hint: Custom Events, Window objects, or a shared state library).
6. **Phantom Dependencies & Workspace Isolation:** Why does a standard `node_modules` setup in a monorepo allow Package A to accidentally import a dependency it didn't declare, but Package B installed? How do strict package managers (like `pnpm`) solve this?
7. **Visual Regression Testing:** Why are traditional DOM snapshot tests brittle for UI? How do pixel-based visual regression tools (e.g., Playwright screenshots, Percy, Chromatic) catch CSS/layout bugs that unit tests miss, and how do you manage "flaky" visual tests?
8. **Module Federation Mechanics:** At a high level, how does Webpack/Rsbuild Module Federation work at runtime? Explain the concepts of `remoteEntry.js`, shared scopes, and version mismatch resolution (e.g., what happens if Host uses React 18 and Remote uses React 17?).
9. **Frontend Feature Flags:** How do you decouple *deployment* from *release* in the frontend? Explain how feature flags prevent broken code from reaching users, and the critical importance of a cleanup process to avoid "flag debt".
10. **The "Boring" Architecture Principle:** Why is a well-structured monolithic frontend (e.g., Laravel + Inertia + Vue/React) often the superior choice for 95% of teams compared to Micro-Frontends or complex monorepos? When is the complexity *actually* justified?
