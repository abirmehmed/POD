# Modern Frontend Architecture: Set 4 (Final)
## SSR, Hydration & Production Optimization

## 📘 10 Conceptual Questions
1. **The Rendering Spectrum (CSR vs. SSR vs. SSG vs. ISR):** Compare Client-Side Rendering, Server-Side Rendering, Static Site Generation, and Incremental Static Regeneration. When does each paradigm make sense for a Laravel-backed application (e.g., marketing pages vs. authenticated dashboards)?
2. **The Mechanics of Hydration:** What exactly happens during "hydration"? Why is it computationally expensive, and what causes a "hydration mismatch" error? How do you safely handle client-only data (like `window.innerWidth` or `localStorage`) in an SSR environment?
3. **Core Web Vitals (CWV) Deep Dive:** How do Largest Contentful Paint (LCP), Interaction to Next Paint (INP), and Cumulative Layout Shift (CLS) directly map to frontend architectural decisions? Why is optimizing for CWV a business metric, not just a technical one?
4. **Bundle Analysis & Chunking Strategies:** How does a bundler (Vite/Webpack) decide what goes into a chunk? Explain the difference between vendor chunks, shared chunks, and route-specific chunks, and how to prevent a single massive `app.js` file.
5. **State Serialization in SSR:** How do frameworks like Inertia.js or Next.js pass data from the server to the client without making a secondary API call? Explain the mechanics of embedding JSON in the initial HTML payload and the risks of serializing complex objects (Dates, circular references).
6. **Islands Architecture vs. Monolithic SPAs:** How does the "Islands" pattern (used by Astro, Laravel Livewire, or htmx) solve the hydration bottleneck by only hydrating interactive components, leaving the rest as static HTML?
7. **Asset Optimization & Modern Formats:** Why should you serve images in WebP/AVIF and fonts in WOFF2? Explain the role of `srcset`, `sizes`, and `font-display: swap` in preventing render-blocking resources.
8. **Progressive Enhancement:** What does it mean to build for progressive enhancement? How do you ensure a core feature (like a search or form submission) works with plain HTML, and then layer JavaScript on top for an enhanced experience?
 is Edge Computing (e.g., Vercel Edge, Cloudflare Workers) changing frontend architecture? How does running SSR or middleware closer to the user reduce latency compared to a centralized origin server?
10. **Long-Term Caching & Cache Busting:** How do hashed filenames (e.g., `app.a1b2c3d4.js`) enable aggressive `Cache-Control: max-age=31536000, immutable` headers? Why must the `index.html` file *never* be cached aggressively?
