# JavaScript Programming Language: Set 4
## Real-Time, Web APIs, and Modern Data Transformation

## 📘 10 Conceptual Questions

1. **Web Components & Shadow DOM:** How do Custom Elements, HTML Templates, and the Shadow DOM work together to create encapsulated, reusable UI components? How does style scoping in the Shadow DOM differ from traditional CSS methodologies (like BEM) or CSS-in-JS, and how do you pierce the shadow boundary when absolutely necessary?
2. **Service Workers & Offline-First:** What is the lifecycle of a Service Worker (install, activate, fetch)? How do caching strategies like "Cache First", "Network First", and "Stale-While-Revalidate" work, and how do they enable Progressive Web Apps (PWAs) to function seamlessly offline?
3. **Web Streams & Backpressure:** How do Web Streams (`ReadableStream`, `WritableStream`, `TransformStream`) handle large data processing without blocking the main thread or exhausting memory? What is the concept of "backpressure," and how does the stream API automatically manage it when the consumer is slower than the producer?
4. **The Temporal API:** Why is the native `Date` object considered fundamentally flawed (mutable, timezone ambiguities, confusing zero-indexed months)? How does the upcoming Temporal API (e.g., `Temporal.PlainDate`, `Temporal.ZonedDateTime`) solve these issues and provide a robust, immutable date/time handling experience?
5. **Regular Expressions Deep Dive:** Beyond basic matching, how do advanced RegExp features work in modern JS? Explain named capture groups (`(?<name>...)`), lookaheads/lookbehinds (`(?=...)`, `(?<=...)`), and the practical difference between the global (`g`) and sticky (`y`) flags.
6. **The `Intl` API & Internationalization:** How does the built-in `Intl` object handle complex locale-specific formatting (dates, numbers, currencies, plurals)? Why is it vastly superior to manual string manipulation or heavy third-party libraries (like Moment.js) for building globally accessible applications?
7. **Resilience Patterns (Retry & Circuit Breaker):** How do you implement robust error handling patterns like "Retry with Exponential Backoff" and "Circuit Breakers" for network requests? How do these patterns prevent cascading failures and increase the resilience of frontend applications when backend services are degraded?
8. **Real-Time Protocols (WebSockets vs. SSE vs. Long Polling):** What are the architectural differences between these real-time communication methods? When should you choose WebSockets (bi-directional, heavy) over Server-Sent Events (uni-directional, lightweight, auto-reconnecting) or Long Polling (HTTP fallback)?
9. **The TC39 Process & Feature Stages:** How does a new JavaScript feature go from an idea to a standardized language feature? Explain the TC39 stages (Stage 0 to Stage 4) and why developers should be cautious about using polyfills or transpilers for pre-Stage 3 features in production.
10. **Modern Data Transformation:** How do modern array and object methods like `Object.groupBy()`, `Array.prototype.flatMap()`, and `structuredClone()` change the way we process data? How do they eliminate the need for older, less readable hacks like `reduce` for grouping or `JSON.parse(JSON.stringify())` for deep cloning?
