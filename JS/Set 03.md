# JavaScript Programming Language: Set 3
## Architecture, Security, and Modern Web Patterns

## 📘 10 Conceptual Questions

1. **Property Descriptors & Immutability:** How do `Object.defineProperty` and accessor properties (`get`/`set`) work under the hood? What are the exact differences between `Object.freeze()`, `Object.seal()`, and `Object.preventExtensions()`, and how do they help enforce immutability in state management?
2. **Modern JS Features (ES2020+):** How do Optional Chaining (`?.`), Nullish Coalescing (`??`), and Logical Assignment (`??=`, `||=`) improve code readability? How does `??` differ fundamentally from the traditional `||` operator when dealing with falsy values like `0` or `""`?
3. **Canceling Asynchronous Operations:** How does the `AbortController` API work? Why is it crucial for canceling `fetch` requests or cleaning up event listeners in modern Single Page Applications (SPAs) to prevent memory leaks and race conditions?
4. **JavaScript Security Vulnerabilities:** Explain Cross-Site Scripting (XSS), Cross-Site Request Forgery (CSRF), and Prototype Pollution. How do you mitigate XSS in vanilla JS (e.g., `textContent` vs. `innerHTML`, Content Security Policy headers), and why is `Object.create(null)` safer for dictionaries?
5. **Module Systems & Bundling Mechanics:** What is the deep difference between ES Modules (ESM) and CommonJS (CJS) regarding synchronous vs. asynchronous loading, and live bindings vs. copied values? How do modern bundlers use ESM for "tree-shaking" to reduce bundle size?
6. **Global Error Handling & Fallbacks:** How do you catch unhandled promise rejections and synchronous errors globally using `window.onerror` and the `unhandledrejection` event? How does this pattern form the foundation of frontend error monitoring tools (like Sentry)?
7. **Observables vs. Promises:** What is the fundamental difference between a Promise (which resolves to a single value and is eager) and an Observable (which emits multiple values over time and is lazy)? How do concepts like "unsubscribing" and "backpressure" apply to data streams?
8. **State Machines & Predictable UI:** How can Finite State Machines (like those used in XState) solve the problem of "impossible states" in complex UI logic? Why is modeling state as a graph with strict transitions safer than using multiple independent boolean flags (e.g., `isLoading`, `isError`, `hasData`)?
9. **Web Performance & The Rendering Pipeline:** How does the browser rendering pipeline work (DOM ➔ CSSOM ➔ Render Tree ➔ Layout ➔ Paint)? How do JavaScript execution, "layout thrashing" (reading and writing DOM repeatedly), and `requestAnimationFrame` impact the Critical Rendering Path and Web Vitals (like INP and CLS)?
10. **TypeScript & The Evolution of JS:** Why did TypeScript become the industry standard for large-scale JS? Explain how structural typing, generics, and utility types (like `Partial<T>` or `Pick<T, K>`) improve architecture and developer experience without changing the actual runtime behavior of the code.
