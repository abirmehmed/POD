# JavaScript Programming Language: Set 6
## Node.js Internals, Tooling, and Full-Stack Architecture

## 📘 10 Conceptual Questions

1. **Node.js vs. Browser Event Loop:** While both use an event loop, they are fundamentally different. How does the Node.js event loop differ from the browser's? Explain the specific phases in Node (Timers, Pending, Poll, Check, Close) and why `setImmediate()` executes before `setTimeout(..., 0)` in Node, whereas the opposite is true in the browser.
2. **SSR, Hydration & "Islands" Architecture:** What is Server-Side Rendering (SSR) and the "Hydration" process? What is the "Uncanny Valley" of hydration (where the UI looks ready but isn't interactive)? How does the "Islands Architecture" (popularized by Astro) solve JavaScript bloat by only hydrating specific interactive components while leaving the rest as static HTML?
3. **Modern Build Tools (Vite, esbuild, SWC):** How did the JavaScript tooling ecosystem evolve from Webpack (written in JS, slow) to modern tools like Vite, esbuild (Go), and SWC (Rust)? What is the fundamental difference between *bundling* (combining files) and *transforming* (transpiling/minifying), and how do these new tools achieve 10x-100x speed improvements?
4. **Testing Pyramid & Mocking Strategies:** What is the difference between Unit, Integration, and End-to-End (E2E) tests? In unit testing, what is the exact difference between a **Spy** (observes calls), a **Stub** (provides fake return values), and a **Mock** (verifies specific interactions)? Why is "mocking everything" considered an anti-pattern?
5. **Client-Side Routing & The History API:** How do Single Page Applications (SPAs) change the URL in the browser without triggering a full page reload? Explain how `history.pushState()`, `history.replaceState()`, and the `popstate` event work together to enable custom client-side routers.
6. **CSS Houdini & The Paint API:** What is CSS Houdini, and how does it expose the browser's rendering engine to developers? Specifically, how does the `Paint API` (using Worklets) allow you to write JavaScript to generate custom, high-performance background images (like dynamic gradients or patterns) that run on a separate thread?
7. **Source Maps & Debugging Minified Code:** When production code is minified and bundled, stack traces become unreadable. How do Source Maps work under the hood (using VLQ encoding) to map a generated line/column back to the original source file? How do error monitoring tools use them to report readable crashes?
8. **The N+1 Query Problem & DataLoader:** In GraphQL or complex REST APIs, what is the "N+1 query problem" (e.g., fetching 100 users, then making 100 separate DB calls for their profiles)? How does the `DataLoader` pattern solve this by batching and caching database requests within the same tick of the event loop?
9. **Accessibility (a11y) & The Accessibility Tree:** How does the browser generate the "Accessibility Tree" from the DOM? Why is semantic HTML always preferred over ARIA attributes? Explain the "First Rule of ARIA" and how JavaScript must manage focus (`element.focus()`) and live regions (`aria-live`) for dynamic content to be screen-reader friendly.
10. **Dependency Injection (DI) & Inversion of Control:** How do you implement Dependency Injection in vanilla JavaScript without a framework? How does passing dependencies into a class via its constructor (rather than importing them directly) decouple your code, make unit testing easier, and enforce the SOLID principles?
