# Rust Programming Language: Set 6 (Applied)
## High-Performance Web Services, WebAssembly & Real-World Integration

## 📘 10 Conceptual Questions
1. **Axum vs. Actix-web Architecture:** Both are top-tier Rust web frameworks. How does Axum’s deep integration with the `Tower` ecosystem (middleware, services) differ from Actix-web’s actor-based origins? Why has Axum become the default choice for many new Tokio-based projects?
2. **Compile-Time Checked Queries (`sqlx`):** How does `sqlx` achieve "compile-time checked SQL" without a heavy ORM macro system like Diesel? Explain how it connects to a live database during `cargo check` to validate queries, and why this is a game-changer for database reliability.
3. **Tower Middleware & The Service Trait:** How does the `tower::Service` trait model request handling in Rust? Explain how middleware (like tracing, rate limiting, or auth) wraps a service, transforming the request/response types before passing them down the chain.
4. **WebAssembly (Wasm) Fundamentals:** What is WebAssembly, and why is Rust considered the best language for writing it? Explain the difference between Wasm's linear memory model and the host environment's memory, and the role of `wasm-bindgen` in bridging the two.
5. **Graceful Shutdown in Async Rust:** When a server receives a `SIGTERM`, why can't it just exit immediately? Explain the mechanics of graceful shutdown in Tokio/Axum: stopping the listener, waiting for in-flight requests to complete, and closing database connections cleanly.
6. **Structured Logging & Observability (`tracing`):** Why is the `tracing` crate preferred over the standard `log` crate in modern Rust? Explain the difference between *events* (something happened) and *spans* (a period of time with a beginning and end), and how they enable distributed tracing.
7. **Async Database Connection Pooling:** How does `sqlx::PgPool` manage connections under the hood? Why is it critical to clone the pool (which is cheap, as it's an `Arc` under the hood) and pass it to request handlers, rather than creating a new connection per request?
8. **Streaming Responses (SSE & Body Streams):** How do you stream data to a client in Rust without loading the entire payload into memory? Explain how `axum::response::sse` (Server-Sent Events) or `tokio_util::io::ReaderStream` handles backpressure and chunked transfer encoding.
Error Mapping (`IntoResponse`):
9. ** How do you elegantly map a custom Rust error enum (e.g., `DatabaseError`, `ValidationError`) to specific HTTP status codes (500, 400) and JSON responses without cluttering the handler logic?
10. **Rust vs. PHP Performance Reality:** When you replace a PHP/Laravel endpoint with a Rust/Axum endpoint, where do the massive performance gains actually come from? (Hint: It's not just "Rust is faster"; think about JIT overhead, GC pauses, process-per-request bootstrapping, and async I/O).
