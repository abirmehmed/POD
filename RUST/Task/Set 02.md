# Rust Programming Language: Set 2
## Advanced Type System, Traits & Error Handling

## 📘 10 Conceptual Questions
1. **Traits vs. Interfaces:** How do Rust `trait`s compare to interfaces in PHP or Java? Explain how traits define shared behavior, and why Rust allows implementing traits for types you don't own (the "orphan rule" and its exceptions).
2. **Generics & Monomorphization:** When you write a generic function like `fn largest<T: PartialOrd>(list: &[T]) -> &T`, how does the Rust compiler handle it under the hood? Explain "monomorphization" and why it results in "zero-cost abstractions" (no runtime performance penalty compared to writing duplicate code).
3. **The `Option<T>` Enum:** Why does Rust not have `null`? How does the `Option<T>` enum (`Some(T)` or `None`) force you to explicitly handle the absence of a value at compile time, eliminating the "billion-dollar mistake" (Null Pointer Exceptions)?
4. **The `Result<T, E>` Enum & The `?` Operator:** How does `Result` model recoverable errors? Explain the mechanics of the `?` operator: how it unwraps `Ok` values, and how it *early-returns* `Err` values, automatically converting error types if a `From` implementation exists.
5. **Smart Pointers: `Box<T>`:** What is a `Box<T>`, and why is it necessary for recursive data structures (like a Linked List or Tree) or when you need to store a large amount of data on the Heap without copying it?
6. **Interior Mutability (`RefCell<T>` & `Rc<T>`):** Rust's borrowing rules are strict at compile time. How do `Rc<T>` (Reference Counted) and `RefCell<T>` allow you to bypass these rules safely at *runtime*? Explain the "Interior Mutability" pattern and when it is appropriate to use it (and the risk of runtime panics).
7. **Static vs. Dynamic Dispatch:** What is the difference between using Generics (`<T: Trait>`) and Trait Objects (`Box<dyn Trait>`)? Explain the performance trade-off between static dispatch (monomorphization, larger binary, faster execution) and dynamic dispatch (vtable lookup, smaller binary, slight runtime overhead).
8. **Closures & Environment Capture:** How do Rust closures (`|x| x + 1`) capture variables from their surrounding scope? Explain the difference between the `Fn`, `FnMut`, and `FnOnce` traits, and when you must use the `move` keyword to force ownership transfer into the closure.
9. **The `From` and `Into` Traits:** Why is implementing the `From` trait the idiomatic way to handle type conversions in Rust? How does implementing `From<Source> for Target` automatically give you the `Into<Target>` implementation for free?
10. **Algebraic Data Types (ADTs):** Rust `enum`s are not just lists of constants (like in PHP); they are Algebraic Data Types. How does attaching data payloads to enum variants (e.g., `Message::Write(String)`) allow you to model complex domain logic safely and exhaustively?
