# Advanced PHP: Set 3
## Concurrency & Asynchronous PHP (Fibers, Swoole, RoadRunner)

## 📘 10 Conceptual Questions
1. **The PHP-FPM Bottleneck:** Why is the traditional "one request = one OS process/thread" model of PHP-FPM inefficient for high-concurrency, I/O-bound applications? How does it lead to memory exhaustion and connection limits under load?
2. **PHP 8.1 Fibers Explained:** What is a `Fiber` in PHP, and how does it enable lightweight, cooperative multitasking? How does it differ from OS-level threads, and why is it the foundational building block for modern async PHP frameworks (like Amp or ReactPHP)?
3. **Event Loops (ReactPHP / Amp):** How does an event loop (using `epoll` on Linux or `kqueue` on macOS) allow a single PHP process to handle thousands of concurrent network connections without blocking?
4. **Swoole / OpenSwoole Architecture:** How does the Swoole extension differ from pure userland async PHP? Explain how it embeds a C-level event loop and coroutine engine directly into the PHP engine, bypassing many userland overheads.
5. **RoadRunner & Persistent Workers:** How does RoadRunner (written in Go) manage a pool of long-running PHP worker processes? Why is this "persistent application" model significantly faster than bootstrapping Laravel on every single request?
6. **Blocking vs. Non-Blocking I/O:** Why does calling a standard blocking function (like `file_get_contents`, `sleep()`, or a synchronous PDO query) freeze the *entire* event loop in an async application? How do you properly perform I/O in this environment?
7. **State Leakage in Persistent Apps:** In RoadRunner or Swoole, the application stays in memory between requests. Why do global variables, static properties, or poorly scoped singletons cause catastrophic state leakage (e.g., User A seeing User B's cart), and how do you properly reset state?
8. **Database Connection Pooling:** Why can't you share a single database connection across multiple concurrent coroutines? How do connection pools solve this by managing a queue of available connections for async tasks?
9. **Error Handling in Coroutines:** How does an unhandled exception inside a Fiber or ReactPHP Promise crash the entire event loop? What are the best practices for isolating and catching errors in concurrent tasks?
10. **The Complexity Trade-off:** Why is traditional PHP-FPM still the correct, pragmatic choice for 90% of CRUD applications? What is the operational and cognitive complexity cost of adopting Swoole, RoadRunner, or async PHP?
